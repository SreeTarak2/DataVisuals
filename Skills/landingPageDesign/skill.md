# Landing Page Generation Prompt

## Overview

This document defines the **complete design and development process for generating a modern SaaS landing page** with:

* smooth animations
* responsive layout
* modern UI patterns
* conversion-optimized structure
* production-ready implementation

The landing page must be **fully complete**, with **no missing sections, placeholders, or unfinished components**.

The system generating the page should act as a **senior product designer and frontend engineer**.

---

# 1. Research Phase

Before generating the landing page, analyze **modern SaaS landing pages (2024–2026)**.

Focus on design patterns used by companies such as:

* Stripe
* Linear
* Vercel
* Notion
* Framer
* Webflow

Identify common patterns including:

* animated hero sections
* scroll-triggered animations
* interactive product previews
* minimal layouts with strong typography
* clear call-to-action placement
* social proof positioning

If modern examples are available, incorporate those design patterns.

---

# 2. Landing Page Architecture

The landing page must include the following **complete sections**.

---

## 2.1 Navigation Bar

Features:

* logo
* product link
* features link
* pricing link
* documentation link
* login button
* primary CTA button

Behavior:

* sticky navigation
* background blur on scroll
* animated hover underline

Responsive behavior:

* hamburger menu on mobile
* animated slide-in mobile menu

---

## 2.2 Hero Section

Purpose: Immediately communicate the product value.

Content:

* bold headline
* short description
* primary CTA
* secondary CTA
* product preview image or dashboard visualization

Animations:

* subtle floating elements
* gradient background motion
* scroll reveal animation
* parallax effects

---

## 2.3 Social Proof

Display trust signals such as:

* company logos
* user statistics
* ratings
* testimonials

Animations:

* auto-scrolling logo carousel
* fade-in testimonial cards

---

## 2.4 Features Section

Layout:

* grid layout (3 columns × 2 rows)

Each feature card includes:

* icon
* title
* short description

Animations:

* hover elevation
* icon motion
* staggered scroll reveal

---

## 2.5 Interactive Product Demo

Provide an interactive preview of the product.

Possible elements:

* dashboard mockup
* animated charts
* tab switching
* hover tooltips

Animations:

* micro-interactions
* chart animations
* smooth transitions

---

## 2.6 How It Works

Explain the workflow in three steps.

Example:

1. Upload your data
2. AI analyzes patterns
3. Receive intelligent insights

Design:

* step-based layout
* timeline style
* animated progress indicators

---

## 2.7 Advanced Capabilities

Highlight advanced product features such as:

* AI insights
* automated dashboards
* predictive analytics
* real-time data analysis

Layout:

* alternating image and text sections

---

## 2.8 Testimonials

Include:

* user avatar
* quote
* company name

Animations:

* carousel slider
* automatic transitions
* fade animations

---

## 2.9 Pricing Section

Include pricing tiers such as:

* Starter
* Pro
* Enterprise

Features:

* feature comparison list
* highlighted recommended plan
* monthly/yearly pricing toggle

Animations:

* hover effects
* card elevation

---

## 2.10 FAQ Section

Accordion layout with:

* smooth expand/collapse animation
* height transition

---

## 2.11 Final Call-To-Action

Large conversion block encouraging the user to start using the product.

Example:

Start analyzing your data with AI today.

Include:

* primary CTA button
* supporting description
* animated background

---

## 2.12 Footer

Include sections for:

* product
* resources
* company
* legal
* social media links

---

# 3. Animation System

Use modern motion design principles.

Micro-interactions:

* button hover effects
* icon animation
* card hover elevation

Scroll animations:

* fade-in elements
* staggered reveals
* parallax motion

Hero animations:

* floating UI elements
* gradient background animation

Recommended technologies:

* Framer Motion
* GSAP
* CSS Scroll Animations

---

# 4. Responsive Design

The page must support all screen sizes.

Recommended breakpoints:

```
320px
480px
768px
1024px
1280px
1440px
```

Mobile optimizations:

* stacked layout
* simplified animations
* large tap targets
* optimized navigation

---

# 5. Performance Optimization

Ensure the page is optimized for performance.

Requirements:

* lazy loading images
* compressed assets
* minimal JavaScript bundles
* optimized animations
* Lighthouse score above 90

---

# 6. Design System

Define a consistent design system.

Colors:

* primary
* secondary
* accent
* background

Typography:

* heading font
* body font
* caption font

Spacing system:

4px spacing scale.

Components:

* buttons
* cards
* icons
* badges

---

# 7. Deliverables

The generated landing page must include:

* complete HTML structure
* full CSS or Tailwind styling
* animation implementation
* responsive design
* production-ready code

No placeholders, incomplete sections, or TODO comments are allowed.

---

# 8. Final Quality Checklist

Before completing the landing page ensure:

* All sections are implemented
* Animations are smooth
* Responsive layout works on all devices
* Code structure is clean and maintainable
* No unfinished components remain
